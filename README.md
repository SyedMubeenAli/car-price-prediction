# 🚗 Car Price Prediction — ML Project

A complete **end-to-end Machine Learning project** that predicts the resale price of a car based on features like year, present price, fuel type, transmission, and more.

Built with **scikit-learn** for modeling and **Streamlit** for the web interface.

## 🚀 Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the Model

```bash
python train.py
```

This will:
- Load and preprocess `data/car_data.csv`
- Train 4 regression models (Linear Regression, Decision Tree, Random Forest, Gradient Boosting)
- Compare performance (R², MAE, RMSE)
- Hyperparameter-tune the best model using RandomizedSearchCV
- Save the final model to `model/model.pkl`

### 3. Run EDA (Optional)

```bash
python notebooks/eda.py
```

Generates visualizations in `notebooks/plots/`:
- Distribution plots
- Correlation heatmap
- Categorical boxplots
- Feature importance chart

### 4. Launch the Web App

```bash
streamlit run app/app.py
```

Open <http://localhost:8501> in your browser to use the prediction interface.

## 📊 Dataset

| Column | Type | Description |
|---|---|---|
| Car_Name | Categorical | Name of the car |
| Year | Numerical | Year of purchase |
| **Selling_Price** | **Numerical** | **Target** — resale price (₹ Lakhs) |
| Present_Price | Numerical | Current showroom price (₹ Lakhs) |
| Kms_Driven | Numerical | Kilometres driven |
| Fuel_Type | Categorical | Petrol / Diesel / CNG |
| Seller_Type | Categorical | Dealer / Individual |
| Transmission | Categorical | Manual / Automatic |
| Owner | Numerical | Number of previous owners |

## 🌐 Deployment

### Option 1: Streamlit Cloud (Recommended)

1. Push the project to a GitHub repository
2. Go to [share.streamlit.io](https://share.streamlit.io)
3. Connect your GitHub repo
4. Set the main file path to `app/app.py`
5. Click **Deploy**

### Option 2: HuggingFace Spaces

1. Create a new Space at [huggingface.co/spaces](https://huggingface.co/spaces)
2. Select **Streamlit** as the SDK
3. Upload all project files
4. The app will auto-deploy

### Option 3: Local Deployment

```bash
streamlit run app/app.py --server.port 8501 --server.address 0.0.0.0
```

## 📝 Key Learnings

1. **Feature Engineering matters** — `Car_Age` (derived from `Year`) is more meaningful than raw year
2. **Drop high-cardinality categoricals** — `Car_Name` with ~100 unique values adds noise with only 302 rows
3. **Ensemble models outperform** — Random Forest and Gradient Boosting beat linear models significantly
4. **Hyperparameter tuning improves results** — RandomizedSearchCV found better parameters efficiently
5. **Save model metadata** — storing feature names alongside the model prevents column mismatch bugs
